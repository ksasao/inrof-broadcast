# 知能ロボットコンテスト 配信テロップシステム

[知能ロボットコンテスト（IRC）](http://www.inrof.org/irc/)を YouTube Live で公式配信する際に、
**TV中継品質のテロップ**を競技の進行に合わせてリアルタイムに切り替えるためのシステムです。
2026年大会で実際の配信に使用しました。

iPhone で撮影した映像を OBS に取り込み、その上に本システムの「ブラウザソース」を重ねて、
ロボットの紹介・対戦カード・結果ボードなどを生放送中にワンクリックで出し分けます。

- **依存ライブラリゼロ**（Python 標準ライブラリのみ）。`pip install` 不要、`python server.py` だけで動く
- 操作はブラウザUIから。配信PCでもタブレットでもOK
- 会場内ローカルネットワークの成績サーバから競技順・結果を取り込み、自動でテロップに反映

## アーキテクチャ概略

```
 ┌───────────┐   映像/音声      ┌────────────────────────────────────────┐
 │  iPhone   │ ───(DroidCam)──▶ │                  OBS Studio              │
 │ (カメラ)   │                 │  ┌────────────────────────────────────┐  │
 └───────────┘                  │  │ ① DroidCam OBS ソース (カメラ映像)  │  │
                                │  ├────────────────────────────────────┤  │
                                │  │ ② ブラウザソース (テロップ・透過)    │◀─┼──┐
                                │  │    http://localhost:8080/overlay    │  │  │ SSE
                                │  └────────────────────────────────────┘  │  │ (即時反映)
                                │      │ 仮想カメラ / 配信                  │  │
                                └──────┼─────────────────────────────────-─┘  │
                                       ▼                                       │
                                 ┌───────────┐                                 │
                                 │ YouTube   │                          ┌──────┴──────────┐
                                 │   Live    │                          │   server.py     │
                                 └───────────┘                          │ (HTTP + SSE)    │
                                                                        │ 状態を保持・配信 │
 ┌──────────────────────┐   POST /api/state                            └──────┬──────────┘
 │ 操作画面 control.html  │ ─────────────────────────────────────────────────▶ │ ▲
 │ (オペレーター/タブレット)│                                                    │ │ POST /api/snapshot
 └──────────────────────┘                                                    │ ▼
                                                                        ┌─────┴───────────┐
 ┌──────────────────────┐  会場ローカルWiFi                             │  obs_snap.py     │
 │ 会場 成績サーバ        │ ◀── fetch_live.py ── build_data.py           │ OBS WebSocket    │
 │ (192.168.x.x)         │      競技順/結果HTML → data/competition.json  │ でカメラ静止画保存│
 └──────────────────────┘                                              └──────┬───────────┘
                                                                              ▼ ws://localhost:4455
                                                                          OBS WebSocket
```

**データの流れは2系統**:

1. **テロップ制御（リアルタイム）**: 操作画面 → `server.py` → （SSE）→ オーバーレイ。
   オペレーターのクリックが即座にOBSの画面に反映される。
2. **競技データ取り込み（随時）**: 会場サーバ → `fetch_live.py` → `build_data.py` → `competition.json`。
   競技順や結果が公開されるたびに取り込み、操作画面を再読み込みすると最新化される。

## 技術要素

| 要素 | 採用技術 | 理由 |
|---|---|---|
| サーバ | Python 標準ライブラリ `http.server`（ThreadingHTTPServer） | 依存ゼロ・会場PCに余計なインストール不要 |
| リアルタイム反映 | **SSE（Server-Sent Events）** | OBSのブラウザソース(Chromium)で確実に動作。WebSocketより実装が単純で、サーバ→画面の一方向push に最適 |
| オーバーレイ描画 | 素のHTML/CSS/JS（フレームワーク非依存） | 軽量・透過背景・1920×1080固定。`rem` を `100vw/1920` 基準にして任意解像度に追従 |
| トランジション | CSSアニメーション（slide/fade、スタッガード） | TV中継風の登場演出。※`clip-path` はOBSのCEFでディセンダ欠けが起きたため不使用 |
| 競技順/結果の解析 | 正規表現ベースのHTMLパーサ | 会場サーバのHTMLは **ISO-2022-JP / EUC-JP / UTF-8 が混在**。エンコーディング自動判別で吸収 |
| カメラ静止画 | **OBS WebSocket v5**（標準ライブラリでクライアント自作） | `SaveSourceScreenshot` でテロップを含まない「カメラ映像のみ」をJPEG保存。外部ライブラリ不要 |
| フォント | Adobe Fonts（Webプロジェクト）/ Google Fonts フォールバック | 後述の「フォントの注意点」参照 |

## ファイル構成とスクリプトの説明

```
inrof-broadcast/
├── server.py            配信サーバ（HTTP + SSE）。操作画面とオーバーレイを仲介
├── build_data.py        競技順/結果HTML → data/competition.json 変換
├── fetch_live.py        会場サーバから競技順・結果・ロボット情報を取得し build_data を実行
├── fetch_sample.py      公式サイトの公開データ(2026)を取得するサンプル取得スクリプト
├── obs_snap.py          OBS WebSocket 経由でカメラ映像のみをJPEG保存（標準ライブラリ実装）
├── public/
│   ├── overlay.html     OBSブラウザソース用テロップ表示ページ（背景透過）
│   └── control.html     オペレーター用 操作画面
├── images/              大会ロゴ logo5.png は初回 fetch 時に公式サイトから取得（git管理外）
├── data/                fetch 実行後に competition.json が生成される（git管理外）
├── docs/
│   ├── wifi-setup.md    会場での2系統WiFi（配信＋成績取得）設定メモ
│   ├── order/           競技順HTML（fetch で取得・git管理外）
│   └── results/         結果HTML・ロボット情報・写真（fetch で取得・git管理外）
└── .claude/launch.json  開発用プレビュー設定（任意）
```

### `server.py` — 配信サーバ
- 状態（現在表示中のテロップ・常時ロゴ・情報カード表示フラグ）をメモリに保持。
- `POST /api/state` で操作画面が状態を更新 → `GET /api/events`（SSE）で全オーバーレイへ即時ブロードキャスト。
- `POST /api/snapshot` で `obs_snap.py` を呼び、カメラ静止画を保存。
- 静的ファイル（public / images / docs / data）も配信。**依存ゼロ**。

### `build_data.py` — データ変換
- 競技順HTML（`ordera`〜`orderd`）を解析し、ラウンド・競技・出場ロボット（2〜3台）に構造化。
  - ヘッダの「競技台」列数を自動検出し、一次予選/敗者復活=3台、二次予選/決勝=2台を正しくグルーピング。
- 結果HTML（`score{c,m}{a-d}`）から競技点・審査点・補正・合計・残り時間を抽出。
- **直前ラウンドの成績**を各ロボットに付与（敗者復活→一次予選、二次予選→敗者復活 or 一次予選、決勝→二次予選）。
- ロボット情報ページ（`rinfo/`）から画像・スペック・アピール文・特徴を抽出。
- ISO-2022-JP / UTF-8 / Shift_JIS / EUC-JP を自動判別。

### `fetch_live.py` — 当日データ取得
- 会場ローカルサーバ（`BASE_URL`）から競技順・結果・ロボット情報＋写真をダウンロードして所定フォルダに保存。
- `--quick` で写真取得をスキップ（競技順＋結果のみ＝数秒）。取得後 `build_data.py` を自動実行。

### `obs_snap.py` — カメラ静止画保存
- obs-websocket v5（OBS 28+ 標準搭載）に**標準ライブラリだけでWebSocket接続**。
- カメラソースを「droidcam」を含む名前で自動検出（`SNAPSHOT_SOURCE` 環境変数で明示も可）。
- `SaveSourceScreenshot` を `imageFormat: jpg` で実行し、**テロップを含まない**カメラ映像のみを保存。

### `overlay.html` / `control.html`
- overlay: ロアサード（ロボット個別）・対戦カード・ラウンドタイトル・得点ルール・スタンバイ・結果ボード（自動スクロール）・常時ロゴ・右上の画像付きロボット情報カードを描画。
- control: ラウンド→競技→ロボットの順に絞り込み、**フォーカス中の1台をワンクリックでオンエア**。並び反転・結果表示・スナップ撮影などの操作。

## セットアップと使い方

> **このリポジトリには競技データを一切含めていません。** まず取得スクリプトを実行して
> データを用意してください（未取得のままだと操作画面に「データ未取得」と表示されます）。

### 1. 競技データを取得（最初に必須）

**公式サイトに公開されている2026年大会の実データ**を取得します（ロボット情報・写真付き）。
会場サーバは不要で、これだけで動作確認できます。

```bash
python fetch_sample.py            # 競技順＋結果＋ロボット情報＋写真をすべて取得
python fetch_sample.py --quick    # 競技順＋結果のみ（高速）
```
取得元: `http://www.inrof.org/2026/irc/score/`。取得後 `build_data.py` が自動実行され、
`data/competition.json` ・ `docs/order` ・ `docs/results`、および大会ロゴ `images/logo5.png`
（公式サイトから初回のみ）が生成されます。

> **サーバ負荷への配慮**: 各リクエストは既定で 0.3 秒間隔を空けて取得します
> （`IRC_THROTTLE` 環境変数で調整可。例 `IRC_THROTTLE=0.5`）。フル取得（写真込み・約160ファイル）でも
> 公式サーバに過度な負荷をかけないようにしています。一度取得したロボット写真・ロゴは再取得しません。

> 取得データは公式サイトで一般公開されているものです。再配布時は出典(inrof.org)を明記し、
> 各データの権利・規約に配慮してください。取得したデータ・ロゴは `.gitignore` で除外されており、
> リポジトリにはコミットされません。

### 2. 起動
```bash
python server.py            # http://localhost:8080
```
- 操作画面: `http://localhost:8080/`
- OBSブラウザソース: `http://localhost:8080/overlay`（幅1920・高さ1080・FPS30・背景透過）
- 別端末から操作: `http://<配信PCのIP>:8080/`

### 3. 当日（本番）のデータ更新
会場ローカルサーバから取得する場合は、環境変数で取得元を指定:
```bash
IRC_BASE_URL=http://<会場サーバ>/2026/irc/score/ python fetch_live.py --quick
```
（Windows PowerShell は `$env:IRC_BASE_URL="..."; python fetch_live.py --quick`）
→ 操作画面を再読み込みすると最新の競技順・結果が反映される。

### 4. カメラ静止画（任意）
OBSの「ツール → WebSocketサーバー設定」で WebSocket を有効化（ポート4455）。
操作画面の「📷 スナップ」で `snapshots/` にJPEG保存（ロボット表示中はゼッケン番号付きファイル名）。

## フォントの注意点（実機で得た知見）

- **Adobe Fonts をPCに同期しても、OBSのブラウザソース（Chromium/CEF）からは font-family 名で参照できません。**
  実機で `Acumin Pro Condensed` / `Ten Mincho Text` が解決不可であることを確認。
  ブラウザでAdobeフォントを使う唯一の方法は **Webプロジェクト（Typekit `<link>`）** の読み込みです。
- 未設定でも Google Fonts（Noto Sans JP / Oswald）に自動フォールバックします。
  Noto Sans JP は源ノ角ゴシックと同一書体のGoogle版なので、和文の見た目はほぼ同等です。
- 詳細は `overlay.html` 冒頭のコメント参照。

## 会場での運営上の注意点

実際の配信運用で重要だった点です。

### ネットワーク（最重要）
- **配信(YouTube)と成績取得(会場ローカルWiFi)で回線を分離**する。配信PCにWiFiを2系統用意し、
  内蔵WiFi＝インターネット（モバイルルータ/レンタル回線）、USBドングル＝会場ローカルWiFi、とした。
- アップロード速度が配信の生命線。**上り4.5Mbps前後あれば720p、ぎりぎり1080p**。
  混雑時に備え 720p / ビットレート2500〜3000kbps を基準にすると安定する。
- 会場ローカルWiFiは**インターネット無し**のため。Windowsのルーティングで
  インターネットが配信回線側に出るよう、会場側アダプタのメトリックを下げる等で固定する。
- 詳細な手順（`netsh` でのSSID固定、同時接続が切れない設定、`#` 始まるSSID(WiFiBOX)の注意）は
  [docs/wifi-setup.md](docs/wifi-setup.md) を参照。

### 当日の進行データ
- 競技順は**各ラウンドの直前**に公開され、ファイル名（`ordera`〜`orderd`）はそのままで中身が差し替わる。
  `fetch_live.py --quick` を都度実行すれば追従できる。
- 結果（特に決勝の審査点）は**最後まで伏せられる**ことがある。合計が「(秘)」表示でも競技点で暫定表示できる。
- 二次予選以降は審査点があり、一次予選・敗者復活戦は競技点のみ、と**ラウンドで列構成が変わる**点に注意
  （本システムは審査列の有無を自動判定して表示を切り替える）。

### 配信オペレーション
- カメラは各チームを随時フォーカスするので、操作画面では**現在の競技の2〜3台だけ**を大きく表示し、
  カメラが向いた台をワンクリックでオンエアできるようにした。
- カメラが競技台の反対側に回ったときのために**並び反転**ボタンを用意した。
- サーバ再起動すると表示状態が初期化されるため、**競技の切れ目やスタンバイ中**に行う。
- `server.py` を変更したときはサーバ再起動が必要。HTML/データ変更だけならブラウザ再読み込みで足りる。
- ポート8080は**1プロセスのみ**起動する（多重起動に注意）。
